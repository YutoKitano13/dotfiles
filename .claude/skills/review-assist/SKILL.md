# PR Review Assist

ユーザーがコードレビューを行う際の補助スキル。AIがレビューするのではなく、ユーザーのレビュー作業を効率化するためのガイドを提供する。

## 入力

- GitHub PRのURL（会話中または `$ARGUMENTS` で渡される）
- トリガーフレーズ: 「レビューをしたい」「レビューします」「PRをレビュー」など

## ワークフロー

### Step 1: 未コミット変更の確認（安全チェック）

**これは最も重要なステップ。絶対にスキップしてはならない。**

```bash
git status --porcelain
```

- 出力が空でない場合 → **絶対にチェックアウトしない**
- ユーザーに以下の選択肢を提示する:
  - `git stash` で一時退避してからチェックアウト
  - 先にコミットしてからチェックアウト
  - レビューを中止
- ユーザーの指示を待ってから次のステップに進む

### Step 2: PR情報の取得

PRのURLから情報を取得する。URLは `https://github.com/{owner}/{repo}/pull/{number}` 形式。

```bash
# PR情報の取得（タイトル、説明、状態、ベースブランチ）
gh pr view <PR_URL> --json title,body,state,baseRefName,headRefName,files,additions,deletions,changedFiles

# PRのdiffを取得
gh pr diff <PR_URL>
```

- **リポジトリの確認**: `gh pr view` が失敗した場合、カレントディレクトリがPRのリポジトリと異なる可能性がある。ユーザーに正しいディレクトリへの移動を案内する。

### Step 3: 作業ブランチへのチェックアウト

```bash
# 現在のブランチ名を記録しておく（後で戻すため）
git branch --show-current

# PRのブランチにチェックアウト
gh pr checkout <PR_URL>
```

### Step 4: 変更内容の深い理解

差分だけでなく、実際のコードファイルを読んで理解する。

1. **変更ファイル一覧を取得**: `gh pr view` の結果から変更ファイルのリストを確認
2. **Readツールで実際のコードを読む**: 変更されたファイルの全体像を把握する
3. **変更ファイルが多い場合（10ファイル以上）**: Exploreサブエージェント（Task tool, subagent_type=Explore）を活用して並列に調査する
4. **関連ファイルの確認**: 変更されたファイルがインポートしているファイルや、変更の影響を受ける可能性のあるファイルも必要に応じて確認する

### Step 5: レビューガイドの出力

以下の形式でユーザーにレビューガイドを提示する。

---

#### 1. PR概要

- **タイトル**: PRのタイトル
- **目的**: PRの説明から抽出した変更の目的（1〜2文）
- **変更規模**: 追加行数 / 削除行数 / 変更ファイル数
- **変更の種類**: 新機能 / バグ修正 / リファクタリング / 設定変更 など

#### 2. 変更の要約

PRで何が行われているかを3〜5行で簡潔に解説する。技術的な背景も含める。

#### 3. 推奨レビュー順序

ファイルを論理的な順序でグループ化し、番号付きリストで提示する。

**順序の決定基準:**
- 型定義・インターフェース → コアロジック → 利用側コード → テスト → 設定ファイル
- 依存関係: ファイルAがファイルBをインポートしている場合、Bを先にレビュー
- 重要度: ビジネスロジックの変更 > ユーティリティ変更 > スタイル変更

**各ファイルの情報:**
- ファイルパス（+追加行数 / -削除行数）
- 変更内容の概要（何がどう変わったか）
- レビューポイント（注目すべき箇所、確認すべき観点）

#### 4. 全体的な注目ポイント（任意）

PR全体を通して特に注意すべき点があれば記載する:
- 破壊的変更の可能性
- エッジケースの考慮
- パフォーマンスへの影響
- セキュリティ上の懸念
- テストカバレッジの十分性

---

## 注意事項

- **AIがレビュー判断をしない**: このスキルはレビューの補助であり、「このコードは問題がある」といった判断は行わない。あくまで変更内容の理解を助け、レビューの効率化を支援する
- **未コミット変更の保護**: Step 1 は絶対にスキップしない。ユーザーの作業を失わないことが最優先
- **日本語で出力**: レビューガイドはすべて日本語で出力する
