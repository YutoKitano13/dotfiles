# Commit & Push

変更内容を確認してからコミットし、リモートにプッシュします。

## 手順

1. `git branch --show-current` で現在のブランチ名を確認する。
2. `git log --oneline -5` で直近5件のコミット履歴を確認し、これまでの作業内容を把握する。
3. `git status` でステージングされていない変更やuntracked filesを確認する。
4. `git diff --staged` でステージング済みの変更内容を確認する。もしステージングされた変更がなければ、`git diff` で未ステージの変更を確認する。
5. 上記の情報をもとに、今回のコミットで何を行うべきかを整理する。
6. 必要に応じて `git add` でファイルをステージングする。
7. コミットメッセージを作成し、`git commit -m "<message>"` でコミットする。
8. `git push` でリモートにプッシュする。

## コミットメッセージの規則

- コミットメッセージは簡潔かつ明確に変更内容を表現すること
- 直近のコミット履歴のスタイルに合わせること
- 必要に応じて conventional commits 形式（feat:, fix:, docs: など）を使用すること

## 注意点

- mainブランチやmasterブランチに直接コミットしようとしている場合は、ユーザーに確認を取ること
- 機密情報（APIキー、パスワードなど）が含まれていないか確認すること
- 大量の変更がある場合は、コミットを分割すべきか検討すること
